CSS

- Cacading Style Sheets, describes how HTML elements are to be displayed.

- separates content design from content, making a website easier to maintain,
more attractive and it saves a lot of work and time.



Layers of a Webpage:

: like how the elements of human body were layered and placed on top
of each other– bones, muscles, veins, etc., to become a whole, the web page,
is also composed of layers to make it whole.

> Content layer

- HTML, the bones.

- defines and layouts the text, images, animation, sound,
video and everything you want to put in a web page with
the use of tags.

- should have:

---> easy to understand text that makes sense without
visual representation, with necessary information for
the environment, like having explaination for abbreviations
and acronyms.
---> unambiguous for the colorblind and text in images needs
to have a sufficient size and contrast.

> Presentation layer:

- CSS, the skin.

- defines how people see your web page in a browser; design.

- should:

---> ensure that text can be zoomed without making the
site unusable, interactive elements of the site are easy
to find, and that images and foreground and background
have enough contrast and are unambiguous to the colorblind.
---> give the site a consistent navigation, aid the user through
business processes, and separate content into easily
understandable units.

> Behavior layer:

- JavaScript, the nerves and the brain .

- real-time user interaction with the web page.

- should:

---> ensure that all the functionality is available to
the user regardless of input device, and make
the user experience as easy as possible by cutting down
on options until they are necessary.

Structure of CSS

What will we use for css?
Curly Braces {}
Colons ::
Semi Colons ;
Selectors (eg. p, h1)
Declerations

Selector - the HTML element you want to style;
is a pattern used to "find" or "select" the HTML elements
on a webpage to which a set of CSS rules will be applied

Property - the style attribute you want to
modify ; is a named characteristic or attribute that
defines a specific aspect of how an HTML element
should be displayed or rendered by the browser.
Properties are used within CSS declarations to apply
styles to selected elements



Value - the weight of the attribute you want to apply
to the element property; is the specific setting or
configuration assigned to a CSS property. A property and
its value form a CSS declaration, which dictates how an
element should be styled

Declaration - is a single instruction that defines
a specific style for an element

Declaration Block - fundamental part of a CSS
rule. It is a block of code that contains one or
more CSS declarations, which define the styling rules
for the selected HTML element(s).




Stylesheet

- A set of instructions to a Web browser on how to display various elements on a Web page is known as style sheet.
- suggests the browser to display a page in a particular way, how the pages should be displayed in the browser.

Every CSS (whether it is contained in a .css file, or embedded in the head element of an HTML document) is a series of instructions called statements .

What does a Statement do?

- identifies the elements in an HTML document that it affects.
- tells the browser how to draw these elements.

Elements are considered to be paragraphs, links, lists items,
and so on. In technical terms HTML terms, an element is anything
marked up inside HTML tags.

Three kinds of CSS

Linked/External Style Sheets
- the most global of the three kinds of CSS because you can
apply the same one to an unlimited number of pages
- allows you to develop a consistent style across pages
Easily allow you to change the layout of your entire
website by simply changing the external style sheet and
every page is instantly updated has a .css file extension

Embedded/Internal Style Sheets
- used for creating a document-wide style rule
- placed within an HTML document, between the head
container tag
- used when you have a page that you want to present
in a different style from other pages
- overrides external style sheets

Inline Style Sheets
- used for isolated changes to a headline, paragraph,
picture or other element
- stored directly in the style attributes of the HTML tags
- inline style sheets override external and embedded style sheets
- overrides external style sheets

Display Property

- allows you to control how an element is displayed



Block Level Element - take up as much space as possible by default.
- each block level element will start a new line on the page, stacking down the page. In addition to stacking vertically, block level elements will also take up as much horizontal space as possible.

Inline Level Element - display in a line . They do not force the text after them to a new line.
- an anchor (or link) is an example of an inline element. You can put several links in a row, and they will display in a line.


Importance:

- how an element behaves when styled with CSS will change based on the display mode of an element (block vs. inline).
- some CSS properties react differently for each display type.







Div Tag - divides the contents of a web page into individual sections
- allows you to apply styles to different sections of your webpage.
- a generic way of adding structure to an HTML document.
- block level element and is therefore, usually applied to block HTML and
should not be used within a paragraph.
- in most web browsers, a div tag will provide line spacing before and after a
block element.











Span Tag - generic way of adding structure to an HTML document
- applied to an inline element.
- used to emphasize words or sentences within a paragraph.

CSS Rule - has two main parts: a selector , and one or more list of style declarations.

Classes
- a user-defined selector.
- used to apply a style rule to specific HTML tags and can control
individual HTML element formatting.
- also a reusable style definition that allows you to apply the
same set of formatting rules to multiple HTML elements



> One element can have more than one class, and one class can be applied to many elements.
> Classes are ideal when you want consistency across multiple elements but still want flexibility.

CSS Selectors - are used to “find” or “select” the HTML elements that you want to style.
- divided into 6 types:

Class selector
Element selector or Tag selector
ID selector
Universal selector
Group selector
Attribute selector

Element Selector
- selects HTML elements (p, div, h1, etc.) and applies CSS to them






ID Selector
- is a style applied to one element in a page. It is normally used once, usually in a page title or the navigation part of the page and defined by the hashtag and uses id attribute of the HTML element. ID selector selects the HTML element with a unique identifier (id) and adds CSS to it. Though, it is not advisable to start an id selector with a number because some browsers do not support it. The ID selector is unique and selects one unique element













Universal Selector
- selects every single HTML element on the page.
- written using the asterisk character













Group Selector
- allows you to select multiple elements and apply the same style to all of them.













Attribute Selector
- selects elements based on specific attribute values. The syntax for the attribute selector is: Element[attribute]













Setting Dimensions
- CSS can be used to manage dimensions such as visibility, width or height of HTML elements.




















Pseudo-Classes and Links
- Recall that classes are user-defined selectors that can be used to control individual HTML element formatting. Pseudo-classes are defined by a colon and are not user-defined. Specific pseudo-classes can be used on specific HTML elements only. The anchor element has four pseudo-classes: link, visited, hover and active. Each one affects the behavior of the anchor element.

> The link pseudo-class is for hyperlinks that have NOT yet visited.
> The visited pseudo-class is for visited hyperlinks.
> The hover pseudo-class is for hyperlinks when the mouse is hovered over them.
> The active pseudo-class is for hyperlinks when they are clicked.
